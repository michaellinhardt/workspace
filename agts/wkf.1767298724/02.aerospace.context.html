AeroSpace

An i3-like tiling window manager for macOS. Fast, keyboard-centric, and designed for power users who want Linux-style window management on their Mac.

View on GitHub Documentation
Quick Install FREE
brew install --cask nikitabobko/tap/aerospace

Key Features

Everything you need for efficient window management on macOS

Tree-based Window Paradigm

Inspired by i3, windows are organized in a hierarchical tree structure with container layout and orientation properties.

Fast Workspace Switching

Zero animations, instant switching without requiring System Integrity Protection to be disabled.

Multi-Monitor Support

Follows i3's paradigm for proper display handling with shared workspaces across monitors.

Plain Text Configuration

TOML-based config files compatible with dotfiles management and version control.

CLI-First Design

Includes manpages and shell completion support. All features accessible via command line.

Multiple Layout Modes

Support for horizontal/vertical tiles and accordion layouts with padding indicators.

Installation

Get AeroSpace up and running in minutes

Homebrew (Recommended)

  1. Download the latest ZIP from the releases page
  2. Unpack the ZIP file
  3. Move AeroSpace.app to /Applications

Configuration

Customize AeroSpace with a simple TOML configuration file

Configuration files are located at ~/.aerospace.toml or ${XDG_CONFIG_HOME}/aerospace/aerospace.toml

~/.aerospace.toml 
# Start AeroSpace at login
start-at-login = true

# Normalizer settings
enable-normalization-flatten-containers = true
enable-normalization-opposite-orientation-for-nested-containers = true

# Main binding mode
[mode.main.binding]
    alt-h = 'focus left'
    alt-j = 'focus down'
    alt-k = 'focus up'
    alt-l = 'focus right'

    alt-shift-h = 'move left'
    alt-shift-j = 'move down'
    alt-shift-k = 'move up'
    alt-shift-l = 'move right'

    alt-1 = 'workspace 1'
    alt-2 = 'workspace 2'
    alt-3 = 'workspace 3'

    alt-shift-1 = 'move-node-to-workspace 1'
    alt-shift-2 = 'move-node-to-workspace 2'

Window Detection Callbacks

Auto-manage windows based on app 
[[on-window-detected]]
if.app-id = 'com.apple.finder'
run = 'layout floating'

[[on-window-detected]]
if.app-id = 'com.spotify.client'
run = 'move-node-to-workspace music'

Keyboard Shortcuts

Default key bindings for efficient window management

Navigation

  • Focus left/down/up/right alt-h/j/k/l
  • Move window alt-shift-h/j/k/l
  • Switch to workspace 1-9 alt-1 to alt-9
  • Move to workspace alt-shift-1 to alt-shift-9

Layout Control

  • Switch to tiles layout alt-/
  • Switch to accordion layout alt-,
  • Resize smaller/larger alt-- / alt-=
  • Toggle fullscreen alt-f

System Commands

  • Enter resize mode alt-r
  • Reload configuration alt-shift-c

CLI Commands

Powerful command-line interface for automation and scripting

Window Management

  • focus - Navigate to windows
  • move - Reposition focused window
  • swap - Exchange window positions
  • resize - Adjust dimensions
  • layout - Switch tiling modes
  • fullscreen - Toggle fullscreen

Workspace Commands

  • workspace - Switch workspaces
  • workspace-back-and-forth - Toggle previous
  • move-node-to-workspace - Relocate windows
  • move-workspace-to-monitor - Transfer workspace

Query Commands

  • list-windows - Output windows
  • list-workspaces - Display workspaces
  • list-monitors - Show monitors
  • list-apps - Enumerate apps
  • config - Query config values

User Verdict

Many users who have tried Rectangle, Amethyst, and Yabai report settling on AeroSpace because it is "nice and stable and gets the job done" without the complexity of Yabai or the limitations of Amethyst.

macOS Settings Recommendations

Optimize your system for the best AeroSpace experience

1

Disable "Displays have separate Spaces"

Run: defaults write com.apple.spaces spans-displays -bool true then log out and back in.

2

Enable "Group windows by application"

In Mission Control settings for better compatibility with AeroSpace.

3

Arrange monitors

Ensure free corner space for hiding inactive workspace windows.

4

Disable Dock auto-hide delay (optional)

Run: defaults write com.apple.dock autohide-delay -float 0 && killall Dock
